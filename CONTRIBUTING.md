# Contribuindo para o Repositório (ENG4021 – Projeto de Software)

Obrigado pelo interesse em contribuir com este projeto! As instruções abaixo orientam estudantes da disciplina **Projeto de Software (ENG4021)** sobre como colaborar de forma organizada, seguindo boas práticas de desenvolvimento.

## 1. Antes de começar

* Certifique-se de ter lido o **README.md** para entender o objetivo do projeto.
* Verifique os **issues** abertos no GitHub e veja se já existe alguém trabalhando naquilo que você pretende fazer.
* Caso não exista um issue relacionado, **crie um novo** descrevendo o problema ou sugestão de melhoria.

## 2. Como iniciar sua contribuição

1. Faça um **fork** do repositório.

1. Clone o fork localmente:

   ```sh
   git clone https://github.com/AlexandreMeslin/ENG4021.git
   ```

1. Crie um **branch** para sua contribuição:

   ```sh
   git checkout -b minha-melhoria
   ```

## 3. Tipos de contribuições aceitas

Você pode contribuir com:

* Correção de bugs
* Melhorias de código ou documentação
* Implementação de novas funcionalidades (após discussão)
* Testes automatizados
* Revisões de código
* Refatorações e melhorias de arquitetura

## 4. Padrões de código

* Siga as convenções das linguagens usadas no projeto.
* Nomeie variáveis e funções de forma clara e significativa.
* Evite duplicação de código.
* Documente trechos complexos.
* Antes de enviar, execute todos os testes (caso existam).

## 5. Commits

* Commits devem ser pequenos e focados.
* Use mensagens de commit descritivas:

  * `fix: corrige erro no módulo X`
  * `feat: adiciona funcionalidade Y`
  * `docs: atualiza documentação`

## 6. Pull Requests (PR)

Para enviar sua contribuição:

1. Faça push do seu branch:

   ```sh
   git push origin minha-melhoria
   ```

1. Abra um **Pull Request** no GitHub.

1. No PR, inclua:

   * O objetivo da mudança
   * Como testar
   * Quais arquivos foram alterados
   * Referência ao issue, se houver (`Closes #12`)
1. Aguarde revisão dos mantenedores.

## 7. Critérios de aceitação

Seu PR será aceito se:

* Resolver um problema real
* Estiver bem explicado
* Seguir o padrão de código
* Não quebrar funcionalidades existentes

## 8. Testes

* Contribuições devem incluir testes quando aplicável.
* Testes devem ser claros e reproduzíveis.

## 9. Trabalhando em equipe

* Mantenha comunicação clara nos issues.
* Respeite a revisão dos colegas.
* Sugestões são bem-vindas durante o review.

## 10. Ambiente de desenvolvimento

Inclua aqui instruções específicas do projeto, como:

* Dependências
* Ferramentas necessárias
* Como rodar o projeto localmente

## 11. Nota para alunos de ENG4021

Contribuir com o repositório faz parte de boas práticas de desenvolvimento adotadas na disciplina. 
Utilize este processo como parte do aprendizado de engenharia de software.
