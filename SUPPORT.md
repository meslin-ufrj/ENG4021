# Suporte e Contato — ENG4021 (Projeto de Software)

Este documento explica como obter suporte ao trabalhar com o repositório da disciplina **Projeto de Software (ENG4021)**, mantido por **@AlexandreMeslin**.

## Antes de pedir suporte

Antes de abrir uma solicitação de ajuda:

1. Consulte o arquivo [**README.md**](README.md).
2. Verifique os arquivos de documentação ([`README`](README.md), [`CONTRIBUTING`](CONTRIBUTING.md), etc.).
3. Busque problemas semelhantes nos **Issues** do repositório.
4. Execute novamente os passos do guia de instalação e execução.

A maioria das dúvidas comuns já está respondida nesses materiais.

## Onde pedir ajuda

Existem três níveis de suporte para os alunos:

### 1. **Issues no GitHub** (preferencial)

Use a aba **Issues** do repositório:

> [https://github.com/AlexandreMeslin/ENG4021/issues](https://github.com/AlexandreMeslin/ENG4021/issues)

Abra um issue quando:

* Encontrar um bug
* Tiver dúvidas relacionadas diretamente ao código
* Precisar relatar falhas na documentação
* Quiser sugerir melhorias

Ao abrir um issue, inclua:

* Descrição clara do problema
* Passos para reproduzir
* Mensagens de erro (se houver)
* Capturas de tela quando apropriado

### 2. **Discussões em sala ou Moodle**

Para dúvidas conceituais, sobre metodologia ou sobre a disciplina em si, utilize:

* Fóruns do Moodle
* Discussões em aula

### 3. **Contato direto com o professor**

Para questões específicas que **não devem** ser públicas, envie e-mail institucional ao professor:

* **Alexandre Meslin** — [meslin@puc-rio.br](mailto:meslin@puc-rio.br)

> Use o contato direto apenas para assuntos pessoais, notas, orientações individuais ou questões administrativas.

## Suporte técnico típico

Os tipos de suporte fornecidos incluem:

* Problemas de instalação ou execução
* Dificuldades para rodar o ambiente de desenvolvimento
* Falhas no uso do Git, GitHub ou workflows

Não incluem:

* Pedidos de solução de tarefas
* Entregas de código por e-mail

## Ajuda com Git e GitHub

Se seu problema for com o uso de Git, tente:

* Rodar `git status`
* Ler o erro completo antes de pedir ajuda
* Descrever exatamente o comando executado

> Veja também na seção de problemas e soluções em [GIT](GIT/README.md)

## Tempo de resposta

* Issues normalmente recebem resposta em **até 48 horas úteis**.
* Respostas podem ser mais rápidas durante períodos de atividades práticas.

## Como formular um bom pedido de ajuda

Inclua sempre:

* **Seu nome completo**
* **Sua turma** (pode ser o hórário se não souber a turma)
* **Seu time** (da disciplina, não de futebol)
* **Contexto**: o que você estava tentando fazer?
* **Estado atual**: o que aconteceu?
* **Comportamento esperado**: o que deveria ter acontecido?
* **Evidências**: logs, prints, mensagens de erro

Exemplo:

> "Ao executar `python manage.py migrate`, recebo o erro X. Meu ambiente é Ubuntu 24.04, instalei dependências com pip. Seguem logs abaixo."
